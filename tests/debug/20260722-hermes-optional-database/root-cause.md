# Hermes 缺失数据库告警

## 用户可感知场景

打开任意项目页面时，未安装或未初始化 Hermes 的环境也会在顶部显示数据库不可用告警。

## 调用链与模块责任

项目概览读取模型扫描 Hermes 本地数据库并返回诊断；项目概览面板渲染非就绪诊断。

## 关键证据

默认 Hermes 路径总会参与扫描；数据库文件不存在时 SQLite 返回 `SQLITE_CANTOPEN`，被转成 `missing` 诊断，前端将其显示为告警。

## 根因与置信度

Confirmed：把可选集成的“未安装/无数据库”当作故障诊断传播到了页面。

## 修复方案

后端静默跳过 `missing` 状态；前端也忽略旧响应中的 `missing` 状态，保留繁忙、损坏及不兼容数据库告警。

## 回归测试

`tests/backend/hermes-session-read-model.test.ts` 覆盖缺失数据库，断言无会话且无诊断。

## 验证结果

通过 Hermes 读取模型测试及前后端 TypeScript 类型检查。

## 阻塞项与剩余风险

无。未启动浏览器进行端到端复查；修复为纯本地读取模型和渲染过滤。
