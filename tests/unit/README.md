# Vitest 快速业务逻辑测试

## 业务场景

`tests/unit` 放由 Vitest 运行的快速业务逻辑测试。适合覆盖共享纯函数、展示前数据归一化、消息去重、排序选择和不需要后端运行态的模块规则。

这些测试必须导入真实源码模块，并使用能对应用户风险的真实业务样例。不要只写组件挂载、框架冒烟或快照测试。

## 运行命令

- `pnpm run test:vitest`：运行 `tests/unit/**/*.test.ts`。
- `pnpm run test:vitest:watch`：开发时监听同一批 Vitest 单测。

## 失败含义

失败通常表示快速业务逻辑已经偏离真实用户场景，例如实时消息重复展示、会话身份匹配错误、展示前数据丢失，或新增测试错误依赖了浏览器、服务端端口、长期进程和全局环境。

## 新增测试

新增测试应优先选择无浏览器、无后端服务、少全局状态的业务模块。如果测试需要真实 API、数据库、WebSocket 服务、浏览器导航或长链路持久化，应放回 `tests/backend`、`tests/spec` 或 `tests/e2e`。
