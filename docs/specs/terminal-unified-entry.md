# 文件目的：沉淀终端统一入口、tmux 保活、显式记录视图和桌面终端布局的长期规格。

## 来源

- `docs/changes/archive/2026-07-06-37-终端统一入口与tmux保活/`

## 需求：会话入口默认进入终端

### 场景：会话卡片默认打开终端并注入恢复命令

当用户在项目总览或工作区左侧点击已有会话卡片时，系统必须打开终端主视图，并向终端注入对应 provider 的恢复命令。

- Codex 会话必须使用真实 `providerSessionId` 恢复。
- Pi 会话必须使用 `pi --session <providerSessionId>` 恢复。
- Claude 会话必须使用 `claude --resume <providerSessionId>` 恢复，并保持 `provider=claude` 身份。
- 默认点击不得直接加载 JSONL 富渲染记录。
- JSONL 记录或详情必须保留为用户显式打开的入口。

### 场景：新建会话走同一个普通终端入口

当用户点击新建会话并选择 Codex、Pi 或 Claude 时，系统必须打开同一个普通终端视图，并注入 provider 对应的新建命令。

- 新建和恢复必须复用终端入口。
- 界面和状态模型不得引入“会话终端”这类额外用户概念。
- 用户退出 TUI 后，终端仍应作为普通 shell 使用。

## 需求：终端由 tmux 保活

### 场景：浏览器断线后终端进程继续运行

当浏览器刷新、网络断开或 WebSocket close 时，后端不得杀死终端进程。真实终端必须继续运行在 `tmux` session 中，浏览器恢复后重新 attach 到同一 session。

- 后端必须能检测、创建、attach 和向 `tmux` session 注入输入。
- 普通 shell 与 Codex/Pi/Claude TUI 必须共享同一保活机制。
- WebSocket close 只能 detach 当前 client，不得 kill 终端进程。
- 只有用户显式结束或删除终端时，才允许 kill 对应 `tmux` session。
- `tmux` 生命周期命令必须真实执行，不能只记录日志或拼接字符串。

### 场景：用户退出 TUI 后仍可执行普通命令

当用户在终端中按 `Ctrl-C` 退出 Codex/Pi TUI 后，终端必须继续留在 shell 中，用户可以运行 `pwd`、`git status` 等普通命令。

- 该行为不得自动更改会话卡片绑定。
- 该行为不得自动切换到 JSONL 记录视图。

## 需求：记录/详情是显式查看入口

### 场景：记录/详情视图由用户显式打开

用户需要查看 JSONL 中的结构化历史、工具调用或消息细节时，必须通过记录或详情入口打开。只有这个入口会加载 JSONL 渲染视图。

- 默认会话点击不加载记录视图。
- 显式点击记录后才渲染 JSONL 历史。
- 记录视图必须能返回终端。
- Claude 默认进入 TUI 时不得请求历史；仅显式记录入口才能读取其分页历史。

### 场景：桌面终端不再固定底部 dock

在桌面视图中，终端必须作为主工作区的平行视图出现，而不是默认固定在底部 dock。

- 页面不得保留可恢复的底部终端 dock 状态。
- 终端必须能填充主工作区。
- 文件面板可以作为辅助视图存在。
- 会话记录/详情与终端是平行切换关系。
