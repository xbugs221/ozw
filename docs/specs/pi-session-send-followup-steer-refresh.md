# 规格：Pi 会话发送、续发、steer 与刷新恢复

> 合并自归档提案：`2026-06-02-58-修复Pi会话发送续发steer刷新稳定性`

## 需求：Pi 日常发送稳定落盘并刷新恢复

### 场景：新建 Pi 会话发送第一条日常需求

- **给定** 用户打开一个真实项目首页
- **当** 用户选择新建 Pi 会话并发送一条日常需求
- **则** 页面应显示用户消息和 Pi 回复
- **且** 刷新网页后两条消息仍可见且不重复

## 需求：Pi 补充需求复用同一会话

### 场景：完成后继续发送补充需求

- **给定** 同一 Pi 会话已经完成第一轮回复
- **当** 用户继续输入补充需求并发送
- **则** 补充需求应进入同一个会话 transcript
- **且** 刷新网页后第一轮和补充轮消息都只出现一次

## 需求：Pi 运行中 steer 状态稳定

### 场景：运行中发送 steer 需求

- **给定** Pi 会话正在处理一个长任务
- **当** 用户在同一会话输入 steer 需求
- **则** 前端应把该输入作为 steer 发送
- **且** 运行态 UI 应显示 Pi queue/steering 状态
- **当** Pi 完成后，steer 用户消息和对应回复应进入持久化 transcript

## 需求：前端实时刷新不重复不丢失

### 场景：实时事件和持久化 read model 收敛

- **给定** Pi runtime 先推送 live 事件，随后写入 transcript 并推送 complete
- **当** 前端收到 complete 并刷新会话消息
- **则** live 副本和持久化副本不得重复显示
- **且** 刷新浏览器后不得丢失任何已完成消息
