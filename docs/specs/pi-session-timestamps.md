# 规格：Pi 会话时间戳展示

## 需求：Pi 会话卡片显示真实 activity 时间

### 场景：历史 Pi 会话不得显示为刚刚

- **给定** 项目存在一个 Pi 会话，最后一条 transcript 事件发生在当前时间两小时前
- **当** 用户打开项目首页查看会话卡片
- **则** 该 Pi 会话卡片应显示“两小时前”语义的时间
- **且** 不得显示“刚刚”

### 场景：未来时间戳不得被误判为刚刚

- **给定** Pi 会话 activity 时间因为单位或时钟问题落在当前时间之后 30 分钟
- **当** 项目首页渲染该会话卡片
- **则** 前端不得把这个时间戳显示为“刚刚”
- **且** 应回落到不误导用户的绝对时间或未知时间

### 场景：Pi 秒级 epoch 时间戳按真实时间展示

- **给定** Pi 会话 activity 时间以秒级 epoch 字符串进入前端
- **当** 项目首页渲染该会话卡片
- **则** 前端应先把秒级 epoch 归一化为毫秒时间
- **且** 按真实历史时间显示相对时间或日期

## 需求：非 Pi 会话时间展示不回退

### 场景：Codex ISO 时间仍按现有相对时间展示

- **给定** Codex 会话提供合法 ISO activity 时间
- **当** 项目首页渲染会话卡片
- **则** 该会话仍按现有相对时间规则展示
- **且** 不会因为 Pi 时间戳修复改变排序字段优先级
