# 规格：项目侧栏注意力导航

### 需求：左侧导航只承担项目导航和项目级提示

ozw 左侧导航必须从会话/工作流子列表职责中收敛出来，只展示项目入口、项目选择、项目级未读或运行态提示，以及必要的项目操作。手动会话、工作流详情、工作流子会话和会话操作由项目主页或工作区内容区承担。

#### 场景：侧栏源码不再拥有会话子列表和会话操作

- **给定** 仓库中的 `frontend/components/sidebar/**` 源码
- **当** 审计左侧导航组件、controller、modal 和公开类型
- **则** `SidebarSessionItem.tsx` 不得存在
- **且** `useSidebarController` 不得维护会话分页、会话点击、收藏、待办、隐藏、删除、重命名逻辑
- **且** `Sidebar.tsx` 和 `SidebarModals.tsx` 不得继续接入会话删除确认
- **且** 侧栏公开类型不得暴露会话或工作流子列表选择/删除 props

#### 场景：侧栏源码不再拥有工作流子列表展开职责

- **给定** 项目中存在运行中、未读或当前选中的工作流/手动会话
- **当** 用户打开项目工作区并查看左侧项目组
- **则** 左侧导航不得维护项目展开状态来展示工作流或会话子行
- **且** `useSidebarController` 不得引用旧的 workflow/session attention 自动展开 helper
- **且** 项目级未读或运行态提示可以保留，但完整列表入口必须留在项目主页或工作区内容区

#### 场景：项目主页继续承载会话和工作流管理

- **给定** 左侧导航已经移除会话和工作流子列表
- **当** 用户从项目清单进入项目主页或项目工作区
- **则** 项目主页仍必须能进入手动会话
- **且** 项目主页仍必须能进入工作流详情或工作流子会话入口
- **且** Web 类型检查必须证明 `ProjectOverviewPanel` 的会话和工作流 props 没有被误删

### 需求：左侧导航动作入口收敛

ozw 必须移除左下角常驻刷新按钮，把项目入口文案改为“打开新项目”，并确保桌面端折叠侧栏后仍能打开项目向导。

#### 场景：展开态移除刷新并重命名打开项目入口

- **给定** 用户在桌面视口打开项目工作区
- **当** 用户查看左侧栏底部动作区
- **则** 页面中不得存在“刷新项目和会话”常驻按钮
- **且** `create-project` 入口必须可见
- **且** 该入口 accessible name 必须是“打开新项目”或 `Open project`

#### 场景：桌面折叠态仍可打开项目向导

- **给定** 用户在桌面视口打开项目工作区
- **当** 用户折叠侧栏
- **则** 折叠态仍必须存在 `create-project` 入口
- **当** 用户点击该入口
- **则** 项目向导必须打开
- **且** 项目向导内部标题可以继续沿用“创建新项目”

### 需求：桌面侧栏宽度跟随内容

桌面展开态侧栏宽度必须从内容需要出发，避免短项目名仍占用旧的固定空白宽度，同时必须设置下限和上限保护可读性与主工作区。

#### 场景：短项目名 fixture 的侧栏宽度小于旧固定空白宽度

- **给定** 用户没有旧的 `ozw:sidebar-width` 本地拖拽宽度
- **当** 用户在桌面视口打开短项目名工作区
- **则** 左侧展开面板宽度必须大于可用最小宽度
- **且** 左侧展开面板宽度必须小于旧的 288px 固定默认宽度
- **且** 旧 `ozw:sidebar-width` 本地值必须被新的 min/max 约束限制
- **且** 移动端抽屉宽度不受该桌面宽度策略影响

### 需求：移动端项目工作区侧栏使用内联折叠布局

移动端进入具体项目会话或工作流后，侧栏必须作为页面内联布局展开，不能再使用遮罩式覆盖抽屉。侧栏展开时主界面仍需可见，用户点击主界面不应关闭侧栏；只有侧栏底部折叠按钮负责收起侧栏。

#### 场景：390px 项目工作区展开侧栏后主界面仍可见

- **给定** 用户在 390px 宽移动视口打开项目工作区会话页
- **当** 用户点击 `Open menu` 展开侧栏
- **则** `project-list` 必须可见
- **且** 聊天主界面必须仍然可见
- **且** 页面中不得出现移动端覆盖抽屉的 `fixed inset-0 z-50` 遮罩容器
- **且** 侧栏宽度必须保留主界面可用空间

#### 场景：移动端侧栏只由底部折叠按钮收起

- **给定** 移动端项目工作区侧栏已展开
- **当** 用户点击聊天主界面
- **则** `project-list` 必须保持可见
- **当** 用户点击 `Hide sidebar` 或 `隐藏侧边栏`
- **则** `project-list` 必须隐藏
- **且** 聊天主界面必须保持可见

#### 场景：移动端底部动作按钮保持紧凑

- **给定** 移动端项目工作区侧栏已展开
- **当** 用户查看侧栏底部动作区
- **则** 打开项目、搜索、设置、折叠侧栏四个按钮必须可见
- **且** 每个按钮高度不得超过 38px
- **且** 每个按钮宽度不得超过 40px
- **且** 相邻按钮水平间距不得超过 8px

#### 场景：移动端首页项目清单仍可进入项目详情

- **给定** 用户在移动端打开 `/workspace`
- **当** 用户查看项目清单
- **则** 项目清单必须可见
- **当** 用户点击 fixture 项目
- **则** 页面必须进入项目详情
- **且** 项目详情页可以继续进入工作流或手动会话
