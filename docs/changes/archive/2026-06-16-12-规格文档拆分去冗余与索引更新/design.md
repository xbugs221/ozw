# 设计：规格文档拆分去冗余与索引更新

## 决策

1. 先建立 `docs/specs/index.md`，再拆长文档，避免拆完后找不到入口。
2. 每篇规格只保留同一业务域的需求和场景。
3. 长文档拆分时保留原文中的强约束，但删除重复矩阵和过期执行叙述。
4. 测试入口是规格的一部分，文档拆分必须同步更新测试注释和 taxonomy。

## 取舍

本提案不追求把所有文档压到极短。目标是让每篇 active spec 可读、可定位、可验证。历史说明可以保留在专门 history/compat 文档里，但不能混在当前主路径规格里。

## 风险

- 拆文档时误删仍有测试覆盖的需求。
- 多个规格引用同一个测试时，索引可能重复但要保持清晰。
- 旧术语清理可能误伤历史兼容边界说明。

## 验证策略

用契约测试扫描文档长度、索引、拆分目标、旧术语和测试入口引用。用 existing regression 保证测试分类说明仍可运行。
