# 提案：回复结束折叠非正文内容

## 为什么要做

长会话里，用户最常回看的内容是智能体最终回复正文。思考和工具调用对审计很重要，但默认全部展开会挤占正文空间，尤其是批量工具调用、多条命令输出和子智能体执行过程。

本次变更在一轮回复完成后，将非正文内容折叠为外层组，只保留正文直接可见。用户需要排查细节时再逐层展开。

## 做什么

```text
流式阶段:
用户消息
  思考流式可见
  工具调用流式可见
  工具输出按现有规则可见/可折叠

进入正文或完成后:
用户消息
  [思考与工具调用  2 个思考 / 3 个工具]  <- 默认折叠
  智能体最终回复正文                         <- 直接可见
```

展开层级：

```text
外层：思考与工具调用
  |
  +-- 思考组
  |
  +-- 工具调用组 A
  |     +-- 命令 1
  |     |     +-- 输出 details
  |     +-- 命令 2
  |           +-- 输出 details
  |
  +-- 批量工具调用组 B
        +-- 命令 1
        +-- 命令 2
```

## 行为规则

| 阶段 | 非正文内容 | 正文 |
|---|---|---|
| 还没开始正文 | 继续流式可见 | 无正文 |
| 正文开始输出 | 正文之前的思考和工具调用自动折叠 | 流式正文直接可见 |
| 本轮完成后 | 非正文默认折叠 | 最终正文直接可见 |
| 用户展开外层 | 显示思考组和工具组 | 正文仍可见 |
| 用户展开工具组 | 显示组内多个工具/命令 | 正文仍可见 |
| 用户展开命令输出 | 显示对应输出 | 正文仍可见 |

## 兼容要求

- 已有工具输出详情折叠继续保留。
- 批量工具调用不能展开成一长串无分组内容。
- 历史消息和 live 消息都遵循同一显示规则。
- `autoExpandTools` 只能影响用户显式调试场景，不应破坏“回复完成后默认折叠非正文”的核心合同。

## 需要更新的旧意图

旧规格中“工具完成后保持展开”的要求，只适用于正文尚未开始的 live 阶段。新意图下，正文开始后应折叠此前非正文内容，但用户仍能逐层展开查看完整细节。
