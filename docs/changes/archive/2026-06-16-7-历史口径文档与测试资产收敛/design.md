# 设计：历史口径文档与测试资产收敛

## 清理边界

允许保留：

- `docs/changes/archive/**` 的历史事实
- 明确用于断言“旧路径不存在”的字符串
- 外部协议或用户历史输入中的原文

不允许保留在活跃入口：

- 把 Codex 当前主路径称为 Codex SDK
- 把 Codex/Pi 一概称为 native SDK
- 把当前 manual chat 描述为 co 文件协议
- 测试文件名与内容明显不符

## 测试资产处置

```
manual/browser-history test
├─ 仍覆盖当前真实行为 -> 迁移到 tests/spec 或 tests/e2e
├─ 只覆盖历史路径 -> 删除或归档说明
└─ 价值不明 -> 标记待确认，不进入主回归
```

## 风险

- 机械替换可能破坏历史语义；必须按上下文分类。
- 删除旧测试可能丢失回归价值；必须先确认是否已有当前测试覆盖。
